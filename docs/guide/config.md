# 配置说明

## certs.conf

路径：`/data/adb/modules/CertBridge/config/certs.conf`

```text
schema_version=2
reqable=1
proxypin=1
```

| 键               | 含义                       | 默认 |
| ---------------- | -------------------------- | ---- |
| `schema_version` | 配置结构版本，请勿手动修改 | `2`  |
| `reqable`        | 启用内置 Reqable CA        | `1`  |
| `proxypin`       | 启用内置 ProxyPin CA       | `1`  |

也可在 WebUI「证书」页用开关修改。内置开关和自定义永久证书仍在重启后生效，避免重写正在使用的开机证书层。

## 安装组件记录

安装脚本会生成 `config/install-profile.conf`，记录默认/自定义安装方式，以及是否安装 WebUI 和免重启热挂载组件。选择不安装 WebUI 不影响开机证书注入；选择不安装热挂载后，`bin/hot_mount.sh` 不会保留在设备模块目录中，WebUI 也会自动隐藏对应区域。

## 自定义证书

适用于本机抓包软件当前使用的根证书与模块内置不一致时（尤其是 **Reqable** 重新生成过 CA；ProxyPin 若未自行换证，通常不必替换）。两张 CA 即使文件大小接近，内容与指纹也可能完全不同，请以 App 内指纹为准。

1. WebUI 支持 PEM（Base64 文本）与 DER（二进制）内容；从用户侧导出时后缀常见 `.pem`、`.crt`（亦可见 `.cer` / `.der`），按内容识别，不必预先改名
2. 设备必须提供 OpenSSL，模块会检查 X.509、有效期和 `CA:TRUE`
3. 模块自动计算 8 位十六进制的 `subject_hash_old` 作为系统信任库文件名，冲突时分配 `.1`、`.2` 等序号
4. 单个文件最大 64 KiB，保存后写入**系统**信任库，重启生效（与「仅装到用户证书」不是同一层）
5. 长期只用自定义证书时，建议在 WebUI 关闭对应内置开关，避免与默认证书并存

也可将证书放入模块目录 `certs/custom/`，下次开机一并合并；日常更推荐走 WebUI，便于校验与管理。

## 临时免重启挂载（热挂载）

把「用户凭据区」或「存储卡上的证书」**临时**提升进系统信任库，**不用重启**即可试抓包；不改永久配置，重启后临时会话自动消失。

### 安装时是否带上

仅在以下情况可用：

- 选择**默认安装**，或
- **自定义安装**时对「免重启热挂载」按音量上确认  

否则设备上不会保留 `bin/hot_mount.sh`，WebUI 也会隐藏热挂载区域，Action 实用菜单会提示组件未安装。需要时请重新刷入同版本并勾选该组件。

### 能做什么

| 模式 | 读取来源 | 说明 |
| --- | --- | --- |
| 用户区 | `/data/misc/user/*/cacerts-added/` | 设置里装到「用户证书」的 CA；会扫所有 Android 用户 / 工作资料 |
| 存储卡 | 默认 `/sdcard/CertBridge`（可改） | 递归扫描 `.pem` / `.crt` / `.cer` / `.der` / `hash.0` |
| 全部 | 用户区 + 存储卡 | 一次合并挂上 |
| 卸载 | — | 只拆本模块本次临时会话，恢复挂载前的系统信任库视图 |

单次最多处理 128 张通过校验的证书（X.509、未过期、`CA:TRUE`、≤ 64 KiB）。

### 怎么用（WebUI）

1. 打开模块 WebUI →「证书」页下方的临时挂载区域  
2. 选模式：用户证书 / 存储卡 / 全部  
3. 若用存储卡：把导出的 CA 放进目录（默认新建 `/sdcard/CertBridge`），或在界面里指定 `/sdcard/`、`/storage/emulated/`、`/mnt/media_rw/` 下的路径  
4. 点挂载 → 看状态是否显示已挂载张数  
5. 不需要时点卸载；或直接重启，临时层也会清掉  

适合：先验证「是不是本机 CA 指纹不对」——导出 Reqable 当前证书放存储卡挂一下，通了再改为 WebUI **自定义证书**（永久，需重启）。

### 怎么用（Action）

模块管理器点 **Action**：

1. 音量下进入「实用功能」  
2. 逐项选择：挂载用户区 / 挂载存储卡（`/sdcard/CertBridge`）/ 挂载全部 / 卸载临时会话  
3. 音量上 = 执行，音量下 = 跳过  

音量上第一级则是刷新权限与状态，不会改挂载。

### 注意

- 临时挂载**不会**改 `certs.conf`、也不会永久写入 `certs/custom/`  
- 「用户区」会把各用户凭据里的 CA 提到**全局系统信任**，请确认那些证书都可信；更稳妥时只用独立存储卡目录  
- 若提示命名空间部分未覆盖：强停目标 App 再开，或卸载后重挂；仍异常可重启  
- 与永久注入可同时存在：简介里会同时提示临时张数与永久证书概况
