# 常见问题

## 刷入后抓包仍证书错误 / 断网？

1. 确认已**重启**
2. Android 14+ 查看状态页是否「运行正常」（APEX 与 system 都会注入）
3. 查看日志中 PID 1、Zygote 是否注入成功
4. **强制停止并重新打开**被抓包的 App（以及 Reqable）；已在运行的进程不会自动继承后来的 Zygote 挂载
5. 确认抓包 App 使用的 CA 与模块启用的证书一致（Reqable / ProxyPin / 自定义）
6. 若显示“证书集合未生成”或“证书注入失败”，查看日志中的实时源数量和命名空间结果

## Reqable 显示「系统证书已安装」但仍抓包断网？

常见原因：

1. **被抓包的 App 进程仍是旧命名空间**：Reqable 自己能看到证书 ≠ 微信/浏览器等已注入。请强停目标 App 后再开，或再重启一次。
2. **证书不一致**：Reqable 每次安装可能生成新 CA。模块内置 `833e2479.0` 若与 App「证书管理」里当前 CA 指纹不同，偶发仍会 TLS 失败。请导出当前 CA 后走自定义导入。
3. **注入后校验过严**：绑定其实已经成功，但内容校验或归属检查在部分命名空间里会误判失败，旧逻辑随后 `umount` 拆掉刚挂上的层，于是出现「App 能扫到证书 / 状态显示正常，TLS 仍失败断网」。当前版本改为：绑定成功即保留，校验失败只记日志；也不再做 bind 后的 `remount,ro`（部分机型上会干扰 Conscrypt 读库）。

## 设置里看不到系统证书？

部分系统对「可信凭据 → 系统」展示有延迟。可先看 WebUI 状态，或重启后再查。

## 与其它系统 CA 模块一起用可以吗？

不建议。证书桥会把开机时已可见的其它模块证书一并复制进本次证书集，但后执行的模块仍可能覆盖先前挂载，无法保证加载顺序，请尽量只保留一个。

## WebUI 打不开 / 无法执行命令？

先确认安装时选择了 WebUI，并检查模块目录是否存在 `webroot/index.html`。已安装 WebUI 时，请用 **KernelSU 管理器** 或其它支持模块 WebUI 的客户端打开；普通浏览器没有 `ksu.exec` 桥接。

## 为什么没有“临时免重启挂载”？

自定义安装时若跳过免重启热挂载，模块不会保留 `bin/hot_mount.sh`，已安装的 WebUI 也会隐藏这一功能，模块 Action 的实用菜单也会提示组件未安装。需要使用时可重新刷入同版本，在自定义安装中按音量上启用，或直接选择默认完整安装。

已安装热挂载时，可在模块管理器打开 **Action**：音量上刷新状态，音量下进入实用功能，逐项选择挂载用户区 / 存储卡（默认 `/sdcard/CertBridge`）/ 全部，或卸载当前临时会话。

## 自定义证书无法上传？

模块要求设备提供 OpenSSL，并拒绝无效、过期、非 `CA:TRUE` 或超过 64 KiB 的证书。文件名无需手动改为 hash。

## 为什么修改开关后没有立即生效？

内置开关和 WebUI 自定义证书属于永久配置。为避免重写正在使用的开机证书层，保存后必须重启。需要免重启时，可使用「临时免重启挂载」读取用户凭据区或存储卡证书。

## 临时证书能完全无痕卸载吗？

正常情况下可以。模块为每次临时会话写入唯一标记，卸载时只对仍显示该标记的命名空间执行一次 `umount`，然后删除临时证书目录；系统文件、用户证书和永久配置均不改变。如果其它脚本在其上继续叠加挂载，模块会拒绝误拆并提示重试或重启。

## 为什么部分已经运行的 App 没有立即信任证书？

模块会尝试覆盖 PID 1、Zygote、抓包 App 与当前可访问的活动挂载命名空间，但部分厂商系统会限制访问个别进程命名空间。WebUI 会显示失败数量；可先强制停止并重新打开目标 App，仍无效时卸载临时会话后重试。

## 为什么提示缺少 nsenter 或 OpenSSL？

临时免重启功能需要 `nsenter`、`stat`、`readlink` 以及支持 `subject_hash_old` 的 OpenSSL。部分 Android 7–8 ROM 或精简 Root 环境不提供这些命令，模块会拒绝热挂载而不是降低校验强度；开机永久注入和 WebUI 其它功能不受影响。

## 为什么不再保存系统 CA 基线？

系统更新、APEX 更新和其它模块都可能让旧基线过期。当前版本在每次重启时直接读取尚未被本模块挂载的实时信任库，生成完整且不可变的本次启动证书集。实时源或生成结果不安全时不会执行挂载，因此无需依赖长期保存的副本。
