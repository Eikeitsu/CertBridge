# 功能介绍

## 做什么

证书桥（CertBridge）把指定 CA 证书**增量挂载**到 Android **系统信任库**。每次开机都从尚未被本模块挂载的实时 system / APEX 证书库读取完整内容，加入模块证书并通过校验后再挂载；模块不保存系统 CA 基线，也不修改系统分区。

## 主要能力

默认安装会启用两张内置 CA 并安装全部能力；选择自定义安装后，可分别跳过 Reqable、ProxyPin、WebUI 或免重启热挂载。

| 能力                   | 说明                                                         |
| ---------------------- | ------------------------------------------------------------ |
| Reqable CA             | 内置 `833e2479.0`，可开关                                    |
| ProxyPin CA            | 内置 `243f0bfb.0`，可开关                                    |
| 自定义证书             | 上传 PEM / DER；自动校验 CA、有效期、hash 与文件名冲突       |
| 开机实时合并           | 每次重启重新读取当前完整 `hash.N` 信任库，不依赖持久化基线   |
| 分版本注入             | Android 7–13 挂载 system；Android 14+ 挂载 Conscrypt APEX    |
| 命名空间验证           | `service.sh` 只补充缺失的 Zygote / Settings 挂载，不重复叠加 |
| 用户证书热挂载（可选） | 读取 `/data/misc/user/*/cacerts-added/`，校验后免重启注入    |
| 存储卡热挂载（可选）   | 递归读取指定目录中的 PEM / DER / CRT / CER / `hash.0`        |
| 无痕卸载（可选）       | 通过会话标记只撤销 CertBridge 临时层，恢复挂载前证书库       |
| WebUI（可选）          | KernelSU / 支持 WebUI 的管理器中配置                         |
| 双层生效               | 永久配置重启生效；用户区 / 存储卡证书可使用临时会话立即生效  |

实时源少于 10 张、复制不完整、附加证书校验失败或挂载后校验失败时，模块会放弃本次注入，系统原始信任库保持不变。

临时会话使用独立的不可变证书目录，不会删除正在挂载的源。挂载覆盖 PID 1、Zygote 和当前可访问的活动挂载命名空间；卸载前同时核对会话标记、源目录设备号 / inode 与 mount ID。若其它模块覆盖在 CertBridge 上方，模块会保留源并要求重试或重启，不会强拆未知层。

## 适用场景

- Reqable / ProxyPin / Charles / mitmproxy 等需要系统 CA 的抓包
- Android 14 及以上系统 CA 放在 APEX 内、普通模块覆盖无效时

## 不做什么

- 不提供抓包代理本身（请使用 Reqable、ProxyPin 等 App）
- 不修改 SELinux 策略以外的系统安全机制
- 不会自动从用户凭据区「搬家」任意证书（请用 WebUI 上传或放入 `certs/custom/`）
